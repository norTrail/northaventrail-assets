# NorthavenTrail.org Website

## Project Overview
Static client-side files (JS, CSS, JSON) for northaventrail.org.
Files are served via Cloudflare CDN at assets.northaventrail.org.
The repo is connected to Cloudflare Pages — committing a file deploys it automatically.

## Pages That use these files
- /hawk-lights
- /where-to-park
- /trailmap
- /map-points-of-interest
- https://northaventrail.org/report-trail-issue
- https://northaventrail.org/adoptgarden
- https://northaventrail.org/tails-2026 (which will move to https://northaventrail.org/tails when the project goes live)
- https://northaventrail.org/trail-captains
- https://northaventrail.org/valentine-cling-map-2027

The goal of these files is to create a common UX / UI for MapBox maps and mapping data across the website. The data for these files (json files) is stored in Google Sheets and generated by Google App Script.

## Important
- Do NOT run a dev server — these are static files only
- No build step required
- Changes go live when committed and pushed to GitHub

## NorthavenUtils Pattern
All scripts reference shared utilities via `window.NorthavenUtils.*` (NOT a direct import).
Guard pattern: `if (window.NorthavenUtils) { ... }` or optional chaining `window.NorthavenUtils?.fn()`.
`northaven-utils.v1.js` must be loaded before all consumer scripts on every page.

## Known Non-Issues (do NOT flag these)
- Squarespace YUI loading warnings — platform-level, cannot fix
- YouTube `compute-pressure` permissions policy warning — third-party embed
- Google Analytics / Doubleclick request failures — ignorable third-party
- Cloudflare Turnstile renders near the submit button (not page top) — this is correct by design
- `mercury:load` event listeners — required for Squarespace AJAX navigation

## Cloudflare Worker Proxy
Located in `cloudflare-worker/worker.js`. Path-based routing:
- POST /submit → Issue Tracker (GAS_ISSUE_URL)
- POST /log → Error Logger (GAS_LOG_URL)
Empty GAS response body is synthesized to {"result":"success"} — this is expected behavior.
